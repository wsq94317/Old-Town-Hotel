# Showcase Goal

## Target
Turn the current prototype into a recordable 3-view showcase build.

The desired recording flow is:
1. Preparation
2. Start Operating
3. Front Desk receives or reassigns a guest
4. Rooms handles cleaning / inspection / reservation actions
5. Lounge handles cup washing or stock recovery
6. End-of-day summary

## Priority Order
1. Front Desk interaction clarity
2. Rooms interaction clarity
3. Lounge interaction clarity
4. Reduce debug feel
5. Improve placeholder-based UI structure
6. Improve recording readiness and flow continuity

## Current View Targets
### Front Desk
- clearly show who is waiting
- clearly show what the guest wants
- clearly show whether a ready room exists
- clearly show what the next action is

### Rooms
- clearly show room state on tile
- clearly show what action fits the selected room
- clearly show worker assignment flow
- keep room interaction visible behind overlay UI

### Lounge
- clearly show stock health
- clearly show washing progress
- clearly show warning / result state
- keep actions grouped and easy to read

## Out Of Scope
- deep backend systems
- review system
- full guest identity
- full economy
- final polished art
- large architecture rewrites
